成员函数cout.put()

成员函数是重要的C++ OOP概念；类定义了如何表示和控制数据，成员函数归类所有，描述了操纵类数据的方法；

在这里，cout.put()的意思是：通过类对象cout来使用函数put()；.被称作成员运算符；

cout.put()提供了一种显示字符的方法，其用法为：

cout.put('$');	 //$

cout.put()的出现是由于C++早期无法正常显示字符常量，为了解决这个问题而设计的；自C++Release2.0之后，cout也可以直接输出字符；

转义序列

如果想要表示单个字符，通常是用单引号''扩起，有些情况下需要输出有特殊含义字符，就要用到转义序列；

  • \n——换行符
  • \t——水平制表符
  • \a——响铃
  • \b——退格
  • v——垂直制表符
  • \r——回车
  • \\——反斜杠
  • \?——问号
  • \'——单引号
  • \"——双引号

关于换行符，有两种版本：一种是字符格式：'\n'，另一种是字符串格式："\n"，其功能是一致的；

转义序列还可以用八进制或十六进制编码表示；将这些编码用''括起来，可以得到相应的字符常量；

在二者都可以使用的时候，应该优先选择字符转义序列，原因在于数字转义序列与一定的编码方式相关，而符号则适用于任何编码方式，可读性也更强；

通用字符名

C++实现支持一个基本的源字符集，即可用来编写源代码的字符集；同时还支持一个执行字符集，其包括程序在执行期间可处理的字符；更进一步地，C++标准还允许实现提供扩展源字符集和扩展执行字符集，C++的这种机制称作：通用字符名（universal character name）；

通用字符名以\u或\U表示，其表示的是字符的ISO 10646码点——可以理解为多文本语言编码的标准化；\u后接8个十六进制位，\U后接16个十六进制位；

如果所用实现支持扩展字符，则可以在标识符中使用通用字符名：

int main()
{
	int k\u00F6rper;//俄文字母
	cout << "let them eat a g\u00E2teau.\n";

	return 0;
}

char的符号

char类型默认情况下，既不是有符号的，也不是无符号的，是否有符号由C++实现决定；如果想要指定char的符号，则可以显式声明为signed char xxx;或unsigned char xxx；

如果将char用作数值类型，则有必要声明指定类型，signed char类型范围为-128~127，unsigned char类型范围为0~255；如果用char存储ASCII码，则无论有无符号，都可以；

wchar_t

程序需要处理的字符集可能无法用一个8位的字节表示，如中文、日文，对于这种情况，C++提供两种方案：

  1. 如果其基本实现字符集为大型字符集，则编译器厂商可以将char定义为一个16位的字节或更长；
  2. 一种实现包含两种字符集，小型基本字符集和较大扩展字符集，而这种较大的扩展字符集就用wchar_t来表示；、

wchar_t类型是一种宽字符类型，也是一种整数类型，它有足够的空间可以表示系统最大的扩展字符集；其与另一种底层类型的长度、属性相同；它的实现依赖于系统，在这种系统上可能为unsigned short，则在那种系统上可能为int；

特别注意的是：wchar_t无法用cout和cin输入输出，因为cout、cin将输出输出视为char类型的流，无法承载wchar_t的输入输出；为此，iostream的新版本中提供了wcin和wcout用于处理wchar_t流；

除此以外，还可以通过前缀L表示宽字符常量和字符串；

wchar_t ch = L'P';//宽字符的表示
wcout << ch << endl << L"P";

char16_t和char32_t

随着Unicode的不断广泛使用，wchar_t逐渐不能满足需求，为此，C++11新增了两种类型：char16_t和char32_t；

其中，char16_t长16位，无符号，使用前缀u表示，与\u00F6形式的通用字符名匹配；

char32_t长32位，无符号，使用前缀U表示，与\U0000222B形式的通用字符名匹配；

char16_t ch1 = u'1';
char32_t ch2 = U'\U0000222B';
cout << ch1 << " " << ch2 << endl;
//49 8747

bool

bool类型是C++的一项新类型——对C++来说；

C++将非零值解释为true，把零值解释为false，通常通过预定义的字面值true、false来表示：

bool isReady = true; //bool类型的声明
cout << isReady; //1

要注意的是：如上输出会输出1，而不是true；

字面值true、false还可以通过提升转换为int类型：

int c = true; //相当与声明 c = 1
int d = false;//相当于声明 d = 0
cout << c << " " << d << endl;
//1 0

同样地，任何数字值和指针值都可以被隐式转换为bool值，任何非零值都被转换为true，零值被转换为false：

bool start = -100;
bool end = 0;
cout << start << " " << end << endl;
//1 0

3.2 const限定符

通常来说，声明一个常量的方式有两种：#define和限定符const；

#define是一个预处理器指令，其声明将其后跟随的字符常量转换成固定的数字，在程序源代码被编译之前进行全文搜索并替换：

#define INT_MAX = 65536

如上，预处理器将在全文搜索INT_MAX并替换为65536；

与#define不同，const是一个限定符，这是一种更好的方式去声明并初始化字符常量：

const int MONTHS = 12;
cout << MONTHS;

字符常量一旦被声明之后将不被允许修改；所以，字符常量声明的时候必初始化，如果将声明和初始化分开，将导致结果的不确定：

const int year;
year = 1998; 
//“year” : 如果不是外部的，则必须初始化常量对象
//“year” : 不能给常量赋值

const较#define先进的地方在于：const可以将定义限制在C++作用域内，如函数或文件中；其二，const可以指定声明类型；

3.3 浮点数

浮点数是C++的第二组基本类型，其表带小数部分的数字；

计算机将浮点数分为两部分存储：其一为基准值部分，如34.145，其基准值为0.34145（不固定，也可以将3.4145看作基准值）；第二部分为缩放因子，如34.145，其缩放因子为100（根据基准值而来，如果3.4145看作基准值，则缩放因子为10）；缩放因子的作用是移动小数点的位置，浮点数因此得名；

浮点数的表示

通常来说，浮点数有两种表示方法：

标准小数点表示法

即：3.1415、214.5236、999.0等

即使小数部分为0，小数点也确保该数字以浮点数格式表示；

E表示法（科学计数法）

即7E5、9.11e-31等等；e前面的部分称之为尾数，e后面的部分称之为指数；指数为负时，表示的不是乘法，而是除法；比如7E5表示的是7*10^5，而9.11e-31表示的是9.11 * 10^-31或者9.11 / 10^31；

通俗的记忆就是：d.dddE+n指的就是将小数点向右移动n位；d.dddE-n指的就是将小数点向左移动n位；

浮点类型

根据可以表示的有效数位和允许的指数最小范围来描述，浮点类型被分为三种，即float、double、long double；

有效位（significant figure）

即数字中有意义的位；和小数点的位置无关；

如：14179 有效位为5；14000 有效位为2； 1.4179 有效位为5；

通常来说，C++中，float有效位为32位，double为64位，long double为80、96或128位；

以下例子展示了本系统中float和double类型的精度：

cout.setf(ios_base::fixed, ios_base::floatfield);
float tub = 10.0 / 3.0;
double mint = 10.0 / 3.0;
const float million = 1.0e6;

cout << "tub = " << tub << endl; // 3.333333
cout << "tub * 1million = " << tub * million << endl;
//tub * 1million = 3333333.250000
cout << "mint = " << mint << endl; //mint = 3.333333
cout << "mint * 1minllion = " << mint * million << endl;
//mint * 1minllion = 3333333.333333

可以直观看到，tub在乘以100w后，从第7个3之后，数值便有了误差，与之对比的double则没有误差；

浮点型的默认值

浮点型常量默认存储为double类型，如果想要指定类型：float则使用后缀f或者F，long double则使用l或者L；

3.4 C++算术运算符

C++提供了五种基本算术运算符：+、-、*、/、%；

其中注意：求模运算符%的操作数（即符号两边的数）只能是整型，用于浮点型将导致编译错误；

运算符优先级与结合性

优先级就是先算谁的问题；结合性就是先从哪算的问题；各大网站都能找到资料，在此不再赘述；

如果不确定自己的代码，请用()；

关于除法

整数相除，结果为整数，且结果向下取整；

浮点数相除，结果为浮点数；如果同为double类型，则结果为double类型，如果同为float类型，则结果为float类型；

cout.setf(ios_base::fixed, ios_base::floatfield);
   cout << "整数相除：" << 9 / 5 << endl; //整数相除：1

   cout << "浮点数相除：" << 9.0 / 5.0 << endl;//浮点数相除：1.800000

   cout << "float类型相除：" << 9.0f / 5.0f << endl; //float类型相除：1.800000

   cout << "float和double类型相除：" << 1e7f / 9.0 << endl; 
//float和double类型相除：1111111.111111

扩展小知识：这里的/运算符其实就是运算符重载的一个例子，即相同符号进行多种运算（int、float、double）；这是OOP的重要属性之一；

关于求模

所谓求模其实就是求余数运算；什么叫余数？就是10 / 4商2余2的余；前文有说过，求模运算符的操作数必须都为整数，否则编译器会报错：

	float xia = 10.0;
   float hua = 4.0;
   cout << xia % hua << endl;

类型转换

C++的类型众多，也给计算机造成潜在风险，为了处理这种情况，C++自动执行类型转换，分为三种情况：

  1. 将一种算术类型的值赋给另一种算术类型的变量时，C++将对值进行转换；
  2. 表达式中包含不同类型时，C++将对值进行转换；
  3. 将参数传递给函数时，C++将对值进行转换；
初始化与赋值进行的转换

C++允许将一种类型的值赋给另一种类型的变量，值将被转换为接收变量的类型：

short x = 200;
long v;
v = x; //short类型被赋值给long类型，值为long类型

cout << v;

在这种情况下，小取值范围的类型赋值给大取值范围类型通常不会发生什么，而将大取值范围类型赋值给小取值范围类型则结果不一定都是好的；

  • 较大浮点类型赋值给较小浮点类型：精度降低，有可能超出范围，结果不确定
  • 浮点类型赋值给整数类型：小数部分丢失，有可能超出范围，结果不确定
  • 较大整型赋值给较小整型：有可能超出范围，通常只复制右边的字节
double p = 3.1415926;
float q; 
q = p;//double赋值给float
cout << q << endl;//3.14159

double m = 3.1415926;
int n; 
n = m;//double赋值给int
cout << n << endl;//3

long x = 3141592657589;
int y; 
y = x;//double赋值给float
cout << y << endl;/*
warning: overflow in conversion from 'long long int' to 
'long int' changes value from '3141592657589' to '1971564213' [-Woverflow]
*/
以{}方式初始化时进行的转换

C++11将{}初始化方式称为列表初始化（list-initialization），列表初始化不允许缩窄（narrowing），即变量的类型肯无法表示赋给它的值；编译器必须知道值能否被类型完整接收，否则将会不通过；

自动类型提升和强制类型提升

当同一个表达式中包含两种不同的算术类型时，C++将执行自动类型转换：

  • 整型提升（integral promotion）：
  1. C++将bool、char、unsigned char、signed char和short值转换为int；
bool b1 = true;
   char ch = 'M';
   cout << b1 + ch << endl;//78 1 + 77 = 78
  1. 如果short比int短，则unsigned short转换为int；如果两种类型长度相当，unsigned short 转换为unsigned int；
  2. wchar_t被提升为int、unsigned int、long或unsigned long中第一个宽度足够存储wchar_t取值范围的值；

  • 表达式中转换

    将不同类型进行算术运算时，C++编译器通过查询校验表来确定算术表达式中执行的转换；

强制类型转换

强制类型转换的设计有两个原因：

  1. 可能有一些值被存储为double类型，但要得到一个

C++支持强制类型转换机制来显式地进行类型转换；有两种方式：

long thron;
   int(thron); //方式1
   (int)thron; //方式2

方式1来自C语言，方式2是纯粹的C++；

注意：强制类型转换不会修改变量本身，而是创建一个新的、指定类型的变量，以供在表达式中使用！

auto声明

auto是C++11重新定义的一个新工具，让编译器能够根据初始值的类型推断变量的类型；称作自动类型推断；

//声明一个int类型变量
auto i1 = 100;//i1为int类型

但auto并非为这种情况而设计，auto被用来处理复杂模型，如STL中的类型；
