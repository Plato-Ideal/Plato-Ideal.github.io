Go语言-函数

本文最后更新于：2021年2月9日 凌晨

函数

函数是组织好的、可重复使用的、用于执行指定任务的代码块

函数定义

Go语言中定义函数使用关键字func；

func sum(x, y int) int { //定义一个函数，
	//参数为两个int类型的变量，返回值为int类型
	return x + y
}

注意：

  1. 函数名：由字母、数字、下划线组成；函数名的第一个字母不能是数字；同一个包内，也不可同名
  2. 参数：参数由参数变量和参数类型组成；多个参数之间用,分隔
  3. 返回值：返回值由返回变量和其变量类型组成，也可以只写返回值的类型，多个返回值必须用()包裹，并用,分隔
  4. 函数体：实现功能的代码块

函数调用

函数的调用非常简单，只需要函数名()这样的方式即可调用函数

func main() { //接上例
	fmt.Println(sum(8, 9))
}

参数

简写

函数的参数如果相邻变量的类型相同，则可以简写参数类型

func addStrings(s1, s2 string) {
	fmt.Println(s1 + s2)
}

因为s1与s2同为string类型，所以s1后面的参数类型可以省略；

可变参数

可变参数指参数数量不固定，可在参数名后加...来标识；

可变参数在函数体中是切片类型

func sum(a int, b ...int) int {
	for _, value := range b {
		a += value
	}
	return a
}

func main() {
	add := sum(1, 2, 3, 4, 5, 6, 7) //a = 1 b = [2 3 4 5 6 7]
	fmt.Println(add)                //28
}

注意：当固定参数和可变参数同时出现，可变参数一定要在最后；也即表明：一个函数声明中最多只能有一个可变参数；

func sum(a ...int, b ...int) int {}

当这样写时，程序会报错：报错信息为：

{
	"message": "can only use ... with final parameter in list",
	"source": "compiler",
}

最后，Go语言中没有默认参数；

返回值

返回值用return；可以不定义返回值，也可以定义了返回值但是不接收；

返回值可以在构造函数的时候直接声明，如果声明了一个返回值变量，则在函数体内，不需要再次声明，直接使用即可，且return后可以不再加变量名；

func intFloat(p, q float64) (sum float64) {
	sum = p + q //sum可以直接使用
	return      //sum可以不加
}

func intFloat2(m, n float64) float64 {
	sum := m + n
	return sum
}

func main() {
	fmt.Printf("直接声明的结果是：%v\n", intFloat(3.14, 9.26))
	//直接声明的结果是：12.4
	fmt.Printf("函数体内声明的结果是：%v\n", intFloat2(3.14, 9.26))
	//函数体内声明的结果是：12.4
}

可以看出，其结果一致，所以此两种方法都可以；

多个返回值

函数可定义多个返回值，需用,分隔，并用()包裹；

func isFlag(name string, age int) (isMarrige, isMale bool) { //定义多个返回值

	if age > 22 {
		isMarrige = true
	} else {
		isMarrige = false
	}
	if name == "Jack" {
		isMale = true
	} else {
		isMale = false
	}

	return
}

func main() {
	fmt.Println(isFlag("Jack", 30)) //true true
}

另，多个返回值也支持简写；

defer语句

Go语言中的defer语句会将其后跟随的语句进行延迟处理；先defer的语句最后执行，最后defer的语句最先执行；

func main() {
	fmt.Println(1)
	defer fmt.Println(2)
	defer fmt.Println(3)
	defer fmt.Println(4)
	fmt.Println(5)
	fmt.Println(6)
	//1 5 6 4 3 2
}

执行规则：

  • 代码的延迟顺序与最终的执行顺序是反向的。
  • 延迟调用是在 defer 所在函数结束时进行，函数结束可以是正常返回时，也可以是发生宕机时。

defer语句的特性决定了它能非常方便地处理资源释放问题：比如：资源清理、文件关闭等

函数进阶

变量作用域

全局变量

声明在函数外面的变量；全局变量只需要在一个源文件中定义，就可以在所有源文件使用；全局变量必须用var声明；如果想要在外部包中使用，全局变量的首字母必须大写；

局部变量

声明在函数内部的变量，作用域只在函数体内；函数的参数与返回值都属于局部变量；

package main

import "fmt"

var a int = 23 //声明全局变量

func main() {

	b := true //声明局部变量

	fmt.Println(a, b)
	for i := 9; i > 5; i-- { //语句块中声明变量
		fmt.Println(a - i)
	}
}

注意：全局变量与局部变量可以同名，函数中使用时优先考虑局部变量；

作用域规则
  • 函数外部访问不到函数内部的变量；如果想要访问，需要用返回值的形式将指定变量返回；
  • 函数内部可以访问到全局变量；
  • 语句块中定义的变量只限当前语句块中使用；外部访问不到；

函数—“一等公民”

Go语言的函数属于“一等公民”（first-clas），也即：

  • 函数本身可以作为值传递
  • 支持匿名函数和闭包
  • 函数可以满足接口
package main

import "fmt"

func add(x, y int) int {
	return x + y
}

func sub(x, y int) int {
	return x - y
}

func calculate(x, y int, op func(int, int) int) int {
	return op(x, y)
}

func main() {
	result1 := calculate(200, 100, add)
	fmt.Println(result1) //300
	result2 := calculate(200, 100, sub)
	fmt.Println(result2) //100
}

在这个例子中，add()及sub()两个函数被作为参数传入；calculate()的参数有三个，两个为int类型， 另一个为func()即函数类型；返回一个int；

匿名函数

匿名函数就是没有函数名的函数，匿名函数多用于实现回调函数和闭包；

func main() {

	//定义匿名函数与使用1
	sayHello := func() {
		fmt.Println("Hello World~")
	}
	sayHello()

	//定义匿名函数与使用2
	func() {
		fmt.Println("hello world!")
	}() //() 表示定义完成立即使用
}

闭包

闭包指的是：一个函数与其相关的引用环境组成的实体；简单来说，闭包=函数+引用环境。

func a() func() { //将函数作为返回值
	name := "Jack"
	return func() {
		say := "Hello"
		fmt.Println(say + name)
	}
}

func main() {
	//闭包就是函数加上外层变量的引用
	r := a() //此时r就是一个闭包
	r()      //相当于执行了a函数内部的匿名函数
}

panic/recover

panic是Go语言中的宕机机制。通过对panic的使用即可手动触发宕机

package main

func main() {

	panic("电脑宕机了")
}

代码运行崩溃并输出：

panic: 电脑宕机了

goroutine 1 [running]:
main.main()
        D:/GO/src/github.com/programer-00/studyGo/day05/panic_recover/main.go:5 +0x45
exit status 2

panic与defer

panic()在宕机时，其后面的代码不会被执行；但是在其前面的defer语句依然会起作用：

func main() {

	defer fmt.Println("1")
	defer fmt.Println("2")
	defer fmt.Println("3")
	panic("电脑宕机了")
}

输出代码如下：

3
2
1
panic: 电脑宕机了

goroutine 1 [running]:
main.main()
        D:/GO/src/github.com/programer-00/studyGo/day05/panic_recover/main.go:10 +0x145
exit status 2

宕机前，defer语句会被优先执行；

recover

当程序宕机了，可以试图通过recover来恢复；

func a() {
	fmt.Println("a")
}

func b() {
	defer func() {
		err := recover()
		if err != nil {
			fmt.Println("func b error")
		}
	}()
	panic("电脑宕机了")
	// fmt.Println("b")
}

func c() {
	fmt.Println("c")
}

func main() { //a func b error c
	a()
	b()
	c()
}
