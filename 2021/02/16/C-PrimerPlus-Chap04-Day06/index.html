4.4 结构

如果要存储的信息是多种类型的，那么很显然，如char数组、int、float等是不能够满足要求的，因为这些变量仅可以存储一种类型的数据；因此，要设计一种数据类型可以存储多种类型数据；于是，结构诞生了；

结构（struct）是一种比数组更为灵活的数据格式，结构为用户自定义类型；

结构创建

创建结构需要两步：

  1. 定义结构描述——即描述结构中可以存储的数据类型；
  2. 创建结构变量（对象）——按照定义创建；
//定义结构描述
struct basketball_player
{
    string name;//球员姓名
    int age;//球员年龄
    string hobby;//球员爱好
};

//创建结构变量
basketball_player r1;
basketball_player r2;

//初始化结构变量
r1.name = "Jackie";
r1.age = 27;
r1.hobby = "running";

其中，结构的各部件名称如下：

  • struct——C++关键字，用来定义结构；
  • basketball_player——标识符，标记成为新类型的名称；
  • string name;——以及其下的两个都是结构成员；
  • {}——开始与结束结构；
  • };——结束结构声明；

在创建结构时，要注意一点：如上例：r1.age指的是结构的age成员；由于age被声明为int类型，因此，r1.age相当于int类型变量；即：r1是一个结构，r1.age是一个int变量，其他成员也同理；

PS：（访问类成员函数的方式是从访问结构成员变量的方式衍生而来）

初始化结构

结构声明位置

对于程序而言，结构声明的位置有两种选择：

  • 外部结构声明：即放在函数外面
  • 局部结构声明：即放在函数内部

C++更提倡使用外部结构声明，因为对于那些函数声明较少的程序来说差别不大，但是对于那些包含两个或更多函数的程序来说，差别非常大：外部结构声明可以被其后的任何函数使用，而局部结构声明只能在其所属函数内部使用；

C++不提倡使用外部变量，但提倡使用外部结构声明；

初始化方式

C++提供多种种声明方式：

struct stu
{   
    string name;
    int age;
    int grade;
    char level;
};	

	//方式1：
	stu s1 = 
    {
        "Jackie Wong",//string name
        18,//int age
        2,//int grade
        'S'//char level
    };
	//方式2：
    stu s2 {"Mary", 15, 8, 'A'};
    stu s3 = {"Pown", 23, 4, 'C'};//= 可选

对于方式2，=是可选的，这种声明方式由C++11支持；

如果按照方式2初始化，且大括号内为包含任何信息，则默认各个成员为零值；

接下来介绍两种更为特别的初始化方式：

可以将定义结构描述及创建结构变量同时完成：

//将定义结构描述和创建结构同时完成
struct mem
{
    string s;
    int i;
} me,you;

甚至可以将初始化也一同完成：

struct member
{
    char ch[20];
    int i1;
} wise = 
{
    "pe",
    12
};

然而，即使如此，还是更推荐将结构定义与变量声明分开，这样可以使得程序更易于阅读和理解；

更甚至你可以连结构名都不写：

//不写结构标识符
struct 
{
    int age;
    string name;
} positon;

这样将创建一个名为position的结构变量，可以用成员运算符来访问成员——position.age；但后果就是无法再创建这种类型的变量；

结构数组

结构数组创建

结构作为一种数据类型，当然也是可以作为元素来创建数组的；方法创建同基本类型数组创建一致；

struct member
{
    char name[20];
    int level;
};

int main() 
{
    member guys[100];//创建一个member类型，名称为guys的数组
}
结构数组初始化

初始化结构数组要先初始化数组，再初始化作为元素的结构：

member girls[2] = //初始化数组
{
    {"Jaccic", 5},//初始化结构
    {"Susan", 6}
};
结构数组使用

结构数组的使用同基本数组类型一致，使用数组索引来访问其中的结构：

cout << girls[0].level << endl;//5
cout << girls[1].name << endl;//Susan

因为girls是一个member类型的数组，所以girls[0]的类型为member，所以可以使用它和句点运算符来访问相应member结构的成员；

4.5 共用体

共用体（union）是一种和机构很相似但又很不同的数据格式；它同样可以存储不同的数据类型；但和结构不同的是，它同时只能存储一种数据类型；

举个例子：

union one
{
    int int_val;
    long long_val;
};

如上，共用体one在使用时，有时是int变量，有时是long变量，但int_val与long_val不能同时存在；

同时，共用体的长度取决于其最大成员的长度；如one的长度就为long_val的长度；

共用体也可以作为结构成员使用：

struct brand
{
    string brand_name;
    int type;
    union id
    {
        long id_num;
        char id_char[20];
    } id_val;
    
};

int main() 
{
    brand app;
    if (app.type == 1)
        cin >> app.id_val.id_num;
    else
        cin >> app.id_val.id_char;
    
}

可以通过层层访问来访问共用体中的成员；

匿名共用体

同匿名结构一样，共用体也同样可以匿名，称为匿名共用体（anonymous union）；匿名共用体没有名称，其成员将称为相同地址处的变量，即每次只有一个成员是当前成员：

struct ano_union
{
    int type;
    union 
    {
        long id_num;
        char id_char[20];
    };
}; 
	ano_union app;

    if (app.type == 1)
        cin >> app.id_num;
    else 
        cin >> app.id_char;

此时，由于共用体为匿名，所以id_num和id_char[20]被视为app的两个成员，它们地址相同，因此，不需要中间标识符；

共用体现在常用于节省内存，如嵌入式等；共用体的另一大功用是操作系统数据结构或硬件数据结构；
