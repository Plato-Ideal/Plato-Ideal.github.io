Go语言-切片

本文最后更新于：2021年2月9日 凌晨

切片

切片（Slice）是一个拥有相同类型元素的可变长度序列，他是基于数组类型做的一层封装，支持自动扩容。

内部结构分为：地址、大小、容量。一般用于快速地操作一块数据集合

切片是引用类型。与数组的值类型不同。

值类型是将自身拷贝一份进行数据传递，备份值改变不会影响自身值

引用类型是将自己的内存地址传递，也就意味着一旦改变——无论是本身还是其他——都会改变数据的值

切片的声明与初始化

声明方式1：直接声明

//切片声明与初始化
var a = []int{1, 2, 2, 3} //切片
fmt.Println(a)
fmt.Printf("a的类型为：%T", a)
var b = [4]int{1, 2, 2, 3} //数组
fmt.Println(b)
fmt.Printf("b的类型为：%T", b)

声明方式2：

既然切片是由数组而来，那么其声明与初始化也可以由数组转换：

//声明方式2：从数组得到切片
var c = [3]int{1, 2, 3}
var d = c[:]
fmt.Printf("d的类型为：%T,d的值为：%v", d, d)

冒号:的左右分别填入数组的索引值，左右都不填代表全部；要注意的是，冒号的左右分别是：左闭右开，即右边的值取不到。

切片的大小与容量

切片的大小表示的是当前元素的数量

切片的容量表示的是底层数组的最大长度

//切片的大小与容量
p := [...]int{2, 34, 5, 6, 7, 7, 8}
q := p[1:5]
fmt.Printf("切片q的大小为：%d\n", len(q)) //切片q的大小为：4
fmt.Printf("切片q的容量为：%d\n", cap(q)) //切片q的容量为：6

注意：切片的容量是从切片的开始直到底层数组的结束，而不是底层数组的全部！！

直接声明的切片，其底层数组即是其本身

m := []int{1, 2, 3, 4, 54, 5}
fmt.Printf("切片m的大小为：%d\n", len(m)) //切片m的大小为：6
fmt.Printf("切片m的容量为：%d\n", cap(m)) //切片m的容量为：6

切片的扩容

运用关键字append来追加；代码展示如下：

a := []int{} //空切片
fmt.Printf("a:%v len:%d cap:%d ptr:%p\n", a, len(a), cap(a), a)
//输出:a:[] len:0 cap:0 ptr:0xa7c578
a = append(a, 1) //将1作为元素传入
fmt.Printf("a:%v len:%d cap:%d ptr:%p\n", a, len(a), cap(a), a)
//输出:a:[1] len:1 cap:1 ptr:0xc0000120b8
a = append(a, 2, 3, 4, 5)
fmt.Printf("a:%v len:%d cap:%d ptr:%p\n", a, len(a), cap(a), a)
//a:[1 2 3 4 5] len:5 cap:6 ptr:0xc00000c2d0
a = append(a, 5, 6)
fmt.Printf("a:%v len:%d cap:%d ptr:%p\n", a, len(a), cap(a), a)
//a:[1 2 3 4 5 5 6] len:7 cap:12 ptr:0xc00001a120

追加策略为：

  • 如果切片大小(len())小于1024个，扩容为原来的1倍，即cap = cap + cap * 1；

  • 如果切片大小(len())大于1024个，扩容为原来的1.25倍，即cap = cap + cap * 1.25；

  • 如果想要扩充的数量为现在容量的2倍，则扩容后的容量为起始值加上扩充的数量；如果扩容后的数量为奇数，则向上追加一个变为偶数；（切片容量总为偶数）

切片的复制

因为切片不同于数组，切片为引用类型，所以不能用简单的赋值操作进行复制

如果直接赋值进行复制，结果如下：

//切片的复制
//切片为引用类型
a := []int{1, 2, 3}
b := a //直接赋值
fmt.Println(b)
b[0] = 100
fmt.Println(a) //[100,2,3]
fmt.Println(b) //[100,2,3]

所以，如果不想影响本身，那么切片不能直接进行复制，需要借助于copy.

a := []int{1, 2, 3}
c := []int{}
copy(c, a) //深拷贝
fmt.Println(c) //[]

此时的c依旧为空的原因是：没有申请内存空间；需要借助make()函数进行内存空间申请

a := []int{1, 2, 3}
c := []int{}
c = make([]int, 3, 3) //申请一块[]int类型，长度为3，容量为3的内存空间
copy(c, a)            //深拷贝
fmt.Println(c)        //[1,2,3]

此时，复制结果准确

切片删除元素操作

Go语言中并没有删除切片元素的专用方法，需要借助切片本身的特性来删除元素

a := []int{1, 2, 3, 4, 5, 6, 7}
a = append(a[:3], a[4:]...) //删除元素4
fmt.Println(a)

通过append来删除特定某项元素的原理是：

将切片按照想要删除的元素为节点分为前后两个部分，前部分作为容器来接收后部分，前后两部分将制定元素留在外面，通过这样的办法来达到删除元素的目的

但是Go语言不支持将切片的切片直接追加，所以需要在后面加...（内置），意为将切片的切片分为一个个元素分别追加

总结

  1. 切片的地址是第一个元素指向的内存空间；其地址改变只有两种情况
    • 切片的首元素改变
    • 容量扩容——原来的内存空间不够用了，重新申请一块，导致内存改变
  2. 切片的容量是跟底层数组息息相关的；当切片再切的时候，其容量从切片的切片的第一个元素开始算；
  3. make是用来给引用类型做初始化的（申请内存空间），对应地，用来给值类型做初始化用new
