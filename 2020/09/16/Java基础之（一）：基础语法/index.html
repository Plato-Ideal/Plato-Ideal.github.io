Java基本语法梳理

标识符

  1. 规则
    • 由26个英文字母大小写，0-9，_或$组成。
    • 数字不可以开头。
    • 不能用关键字或者保留字，但能包含关键字或者保留字。例如：public不可以，但publicabc则是可以的。
    • Java严格区分大小写，但对长度无限制。
    • 标识符不能包含空格。
  2. 规范
    • 包名：都小写，如inheritancetest
    • 类名、接口名：所有单词首字母均大写，CheckAccount（大驼峰规则）
    • 变量名、方法名：第一个单词首字母小写，其余单词首字母均大写，annualInterestRate(小驼峰规则)
    • 常量名：所有单词都大写，AMOUNT

变量

  1. 概念

    • 内存中的一个存储区域，该区域的数据可以在同一类型范围内不断变化，变量是程序中最基本的存储单元，包括变量类型、变量名和存储的值

  2. 注意点

    • Java中每个变量必须先声明，后使用
    • 使用变量名来访问这块区域的数据
    • 变量的作用域：其定义所在的花括号内
    • 变量只有在其作用域内才有效
    • 同一作用域内，不能定义重名的变量

  4. 另一种分类方式：

    成员变量

    实例变量（不以static修饰）

    类变量（以static修饰）

    局部变量

    形参（方法，构造器中定义的变量）

    方法局部变量

    代码块局部变量

  • 异同：
    • 异：局部变量形参除外，需显式初始化
    • 同：都有生命周期

基本数据类型提升

  1. 前提：只是七种基本数据类型变量间的运算，不包含boolean

  2. 分类

    • 自动类型提升

      • 容量大的数据类型变量与容量小的数据类型变量做运算时，结果自动提升为容量大的数据类型
      • 说明：容量大与小不是说占用内存的多与少，而是数的范围的大小
      • 补充：char与byte、short类型做运算时，都需要由int类型来接收（结果都为int型）

    • 强制类型提升

      • 自动类型提升运算的逆运算
      • 格式：
      double d1 = 12.9;
int i1 = (int)d1;//截断操作
      • 需要强转符()
      • 强制类型转换可能导致京都损失

  3. 归纳总结：

    • 整形常量默认为int型，浮点型常量默认为double型
    • 无论什么情况，long型常量和float型都要在其后加L和F

运算符

  1. 定义：

    • 运算符是一种特殊的符号，用以表示数据的运算、赋值和比较等

  2. 分类：

    • 算术运算符

      • 正负号 + ，-

      • 四则运算 +，-，*，/

      • 取模 %

        取模运算的正负性与被模数相同，开发中常用来判断能否除尽的情况

      • 自增 ++i，i++：前者先自增后赋值，后者先赋值后自增

      • 自减 —i，i— ：前者先自减再赋值，后者先赋值后自减

        注：自增和自减运算不改变变量本身的数据类型

      • 字符串连接：+

    • 赋值运算符

      • 符号：=

      • 当 = 两边数据不一致时，可以使用自动类型提升或使用强制类型转换原则进行处理

      • 扩展赋值运算符：
        +=、-=、*=、/=、%=

        不会改变变量本身的数据类型

      • 开发中，实现➕2的操作方法：

        int num = 1;
num = num + 1;//2
        int num = 1;
num += 1;//2(推荐)

      • 开发中，实现➕1的方法：

        除以上两种外：

        int num = 1;
num++；//2

    • 比较运算符（关系运算符）

      • == , != , < , > , <= , >= ,instanceof(检查是否是类的对象)

    • 逻辑运算符

      • & 逻辑与

        && 短路与

        • 区别&和&&：

          相同点1：& 和 && 运算结果相同

          相同点2;当符号左边为TRUE时， 二者都会执行符号右边的运算

          不同点：当符号左边为FALSE时，&会执行符号右边的运算，&&则直接返回FALSE

      • ｜ 逻辑或

        ｜｜短路或。

        • 区分｜和｜｜“｜：

          相同点1：二者结果相同

          相同点2:左边都为FALSE时，二者都会执行符号右边的语句

          不同点：当左边为TRUE时，|会执行右边语句，||不会执行

      • !逻辑非

      • ^逻辑异或

    • 位运算符

      • 位运算符是直接对整数的二进制进行的运算，操作的都是整型的数据
      • << 左移 ：在一定范围内，每左移一位相当于乘一次2
      • >> 右移：在一定范围内，每右移一位相当于除一次2
      • >>> 无符号右移：被移位二进制最高位无论是0还是1，空缺位都用0补齐
      • &：与运算：二进制位进行&运算，只有1&1结果是1，否则是0
      • |：或运算：只有0|0结果是0，否则是1
      • ^：异或运算：相同二进制位进行^运算，结果是0，不相同则为1
      • ~：取反运算：正数取反，各二进制码按补码各位取反，负数取反，各二进制码按补码各位取反

    • 三元运算符：

      • 格式：

        //(条件表达式1)?表达式1:表达式2;

      • 说明：

        • true：结果运算为表达式1
        • false：结果运算为表达式2
        • 表达式1和表达式2不一定为同一种类型，但一定可以统一到一种类型上去
        • 三元运算符可以嵌套使用
        • 凡是可以使用三元运算符的地方，都可以使用if-else；反之，不成立
        • 如果程序即可以使用三元，又可以使用if-else，优先选择三元运算符 原因：简洁、效率高

      • 三元运算符与if-else的联系和区别

        • 三元运算符可以简化if-else语句
        • 三元运算符要求必须返回一个结果
        • if后的代码块可有多个语句

  3. 优先级

    • 只有单目运算符、三元运算符和赋值运算符是从右向左算的
    • 开发中，想让谁先算就加个小括号（）
